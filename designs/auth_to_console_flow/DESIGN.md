# Auth to Console Flow — DESIGN.md

> 阶段：Inception | 状态：待审阅 | 推荐 Skill：/login + /dashboard + /list-page
> 创建：2026-05-15

---

## 1. 需求摘要

**目标用户：** Zenlayer zenConsole 云控制台用户（IT 管理员 / 开发者）  
**使用场景：** 用户首次或日常访问控制台，完成身份认证后进入主界面，再导航至资源列表  
**核心任务：** 展示完整的「登录 → Dashboard → 列表页」用户旅程，并在各屏间标注跳转关系和关键交互说明

---

## 2. 信息架构

```
┌─────────────────────────────────────────────────────┐
│  Screen 1 — Login Page                              │
│  · 邮箱 + 密码输入 / Sign In 按钮                  │
│  · 支持 MFA、忘记密码、SSO 入口                    │
└──────────────────┬──────────────────────────────────┘
                   │ ① 登录成功 → 跳转
                   ▼
┌─────────────────────────────────────────────────────┐
│  Screen 2 — Dashboard                               │
│  · 资源概览卡片（用量 / 费用 / 告警）               │
│  · 快捷操作区                                       │
│  · 近期活动 / 公告栏                               │
└──────────────────┬──────────────────────────────────┘
                   │ ② 点击资源类型 → 跳转
                   ▼
┌─────────────────────────────────────────────────────┐
│  Screen 3 — Resource List Page                      │
│  · 顶部 Tab / 筛选栏                               │
│  · 表格：状态、名称、地区、规格、操作列             │
│  · 分页 / 批量操作                                 │
└─────────────────────────────────────────────────────┘
```

**标注说明（需在 Figma 上添加）：**
- ① 「登录成功」箭头：从 Login Screen 的 Sign In 按钮指向 Dashboard
- ② 「进入列表」箭头：从 Dashboard 资源卡片指向 List Page
- 各屏标注：页面名称 + 触发条件 + 目标页
